# Naval War + Coastal Reliability Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add amphibious assault (embarked melee storms the shore, disembarking on capture — for human and AI), make the AI value coastal cities (reliable naval bootstrapping), and teach the AI to invade sea-only enemy coastal cities.

**Architecture:** One validation-rule change + a data-driven attack penalty (the melee resolver already disembarks the attacker on capture). Two AI changes: a coastal bonus in `knownGoodSpots`, and a `seaPath` advance fallback in `campaignOrders` (the amphibious `ATTACK` then auto-fires via the existing `navalFight` once the embarked unit is adjacent). All AI logic is a pure fn of (state, fog).

**Tech Stack:** TypeScript, Vitest. Deterministic engine; self-play replay is the gate.

**Spec:** `docs/superpowers/specs/2026-06-19-naval-war-design.md`

---

## File Structure

- `src/engine/validate.ts` — narrow the embarked-attack block (permit melee-vs-shore).
- `src/engine/systems/combat.ts` — `attackStrength` gains `amphibious?`; the two melee call sites pass `isEmbarked(...)`.
- `src/data/standard/index.ts` — `settings.naval.amphibiousAttackPct: -33`.
- `src/ai/economy.ts` — coastal bonus in `knownGoodSpots`.

- `tests/islands-mapgen.test.ts` — island invasion behavioral test.
- `tests/selfplay.test.ts` — re-seed 314/960 only if the gate shows they flipped.

**Task order:** Task 1 (rule) + Task 2 (penalty/capture) are the engine mechanic. Task 3 (coastal valuing) and Task 4 (invasion AI) are the AI. Task 5 is the behavioral test + re-tune gate.
